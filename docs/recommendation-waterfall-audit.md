# Recommended for You waterfall audit

## Summary

`Recommended for You` is generated by `fetchPersonalRecommendations` in `src/services/personalRecommendationsService.ts`. The old pipeline had one personalized query (`top genres` + optional Metacritic), then cached that small result set. If RAWG returned owned/wishlisted games, genre-diversity removed too many games, or the query was overly narrow, the Home/Discover surfaces were quickly backfilled by the separate Trending feed. That made Trending look like the primary strategy.

The revised pipeline is a progressive personalized waterfall. It tries several personal sources, records why candidates are accepted or rejected in development logs, relaxes thresholds before fallback, and only asks for RAWG Trending after the personalized pool remains below the floor.

## Current waterfall

Target size: 12 candidates. Personalized floor before Trending: 10 candidates.

1. **Games similar to highly rated / finished / currently playing games**
   - Source: RAWG `/games/{id}/suggested` for up to five strongest positive library games.
   - Selection reason: `Playing`, high-rated `Finished`, or otherwise strong positive recommendation signals.
   - Rejection reasons: duplicate RAWG id, already owned/wishlisted/finished/dropped, score below strict similar-game threshold.

2. **Strict affinity match**
   - Source: RAWG `/games` filtered by the user's top three genres, top tags, and a softened Metacritic floor when the user has a high Metacritic preference.
   - Selection reason: strongest genre/tag affinities from finished, playing, planned, wishlist, and high-playtime games.
   - Rejection reasons: duplicate, owned/wishlist, low score, later genre/source diversity.

3. **Platform Plans and Wishlist interests**
   - Source: RAWG suggested games for wishlist and `Want to play` titles.
   - Selection reason: explicit future intent.
   - Relaxation: lower score threshold and wider scoring inputs than strict affinity.

4. **Recently interacted games**
   - Source: RAWG suggested games for recently played library games.
   - Selection reason: recent activity.
   - Relaxation: same first relaxation tier as plans/wishlist.

5. **Other personalized candidates via progressive relaxation**
   - Source: wider top-five genre query, expanded top-eight tag query, and a popularity-ordered genre query.
   - Relaxation: lower minimum score; scoring considers more genres/tags/platforms and allows a wider Metacritic distance.

6. **Second-order recommendations**
   - Source: RAWG suggested games from the best personalized candidates already found.
   - Selection reason: personalized expansion from candidates related to the user's profile, not generic Trending.
   - Relaxation: broadest personalized threshold before fallback.

7. **RAWG Trending fallback**
   - Source: RAWG `/games` ordered by `-added`.
   - Trigger: only after all personalized stages still produce fewer than 10 candidates.
   - Cap: when any personalized candidates exist, Trending can add at most two candidates and is labeled `Trending fallback`.

## Diagnostics

Set `VITE_DEBUG_RECOMMENDATIONS=true` in a development build to enable console diagnostics. The grouped debug output includes:

- candidate source
- RAWG id and title
- similarity score
- acceptance/rejection decision
- rejection reason (`duplicate`, `owned`, `wishlist`, `finished`, `dropped`, score threshold, genre diversity, source diversity)
- stage production counts
- fallback consideration and fallback selection reasons
- final personalized vs. Trending counts

## Expected behavior

Users with a reasonably sized and enriched library should usually see a mixed section: similar-to-liked games, genre/tag affinity picks, plan/wishlist-related games, recent-game expansions, and possibly a small Trending tail. A full row of Trending should only occur when personalized sources are genuinely unavailable or exhausted.
